# Pictoria i18n 设计（中文 + 英文）

日期：2026-06-05
状态：已采纳（goal 模式下自主决策，依据 brainstorming 流程对比方案后选定）

## 目标

前端 UI 全量国际化，至少支持简体中文与英文；tag 显示名的本地化跟随 UI 语言；
数字/日期格式化跟随 locale；语言可在 Settings 中切换并持久化。

## 现状

- 前端无任何 i18n 库；UI 文本中英混杂（英文为主，侧边栏/撤销文案等为中文）。
- `vite.config.ts` 的 AutoImport `dirs` 已预留 `./src/locale`（目录不存在）。
- 后端 `services/tag_i18n.py` 已有 `tag.zh-Hans.json` / `tag.ja.json` 两张翻译表，
  但 `translate_tag` 的 `lang` 固定默认 `zh-Hans`，端点不接受语言参数。
- 数字格式化硬编码 `Intl.NumberFormat('en-US')`（App.vue、BottomBar、TreeList、
  PostMultiSelectPanel）；日期用裸 `toLocaleString()`。

## 方案对比

| 方案 | 结论 |
|------|------|
| **A. vue-i18n（Composition mode）+ 集中式 TS 消息目录** | ✅ 采纳。成熟、插值/复数/回退齐全；TS 消息文件使 zh/en key 对齐由 vue-tsc 在编译期强制 |
| B. 自研轻量 composable | 否。需重造插值/复数；扩展第三语言成本高 |
| C. vue-i18n + SFC `<i18n>` 块 | 否。翻译散落 50+ 组件，无法整体审阅双语对齐 |

## 设计

### Locale 标识

- 支持 `en`、`zh-Hans`（与后端翻译表命名一致）；设置值另有 `auto`。
- `auto` 按 `navigator.languages` 解析：`zh*` → `zh-Hans`，否则 `en`。
- `fallbackLocale: 'en'`。
- 持久化：`localStorage['pictoria-locale']`，默认 `auto`（与 `pictoria-color-scheme` 同模式）。

### 模块结构

```
web/src/locale/
  index.ts            # createI18n 实例、localeSetting/resolvedLocale、
                      # html lang 同步、formatNumber/formatDateTime helpers
  messages/
    en.ts             # 英文目录 = schema 源（export type MessageSchema = typeof en）
    zh-Hans.ts        # 中文目录，类型标注为 MessageSchema → key 缺失/多余编译报错
```

- `index.ts` 位于 AutoImport 扫描目录，导出的 `localeSetting`、`resolvedLocale`、
  `formatNumber` 等可在组件中直接使用；`messages/` 子目录不被 AutoImport 扫描。
- `useI18n` 通过 unplugin-auto-import 的 `vue-i18n` preset 自动导入。

### 使用约定

- 模板内静态文本：`$t('key')`（vue-i18n globalInjection）。
- `<script setup>` 内逻辑文本：`const { t } = useI18n()`。
- 非组件模块（shared/mutations.ts、undoSnackbar.ts、useAPIError.ts）：`i18n.global.t`。
- 静态选项数组（排序选项、评分桶等）：存 message key，渲染处再翻译，
  保证切换语言即时生效。
- 复数：vue-i18n 管道语法（`'{n} post | {n} posts'`），中文给单一形式。

### Key 命名空间

`common`（取消/删除/加载中/搜索/撤销…）、`nav`、`sidebar`（目录树+操作菜单+删除对话框）、
`settings`、`filter`（含 rating/score/silva/waifu/extension/tags 子域）、`sort`、
`rating`、`post`（详情面板+查看器）、`multiSelect`、`bottomBar`、`import`、
`tagsView`、`tagSelector`、`history`（撤销系统文案）、`dropOverlay`、`error`。

### 数字/日期

- `locale/index.ts` 导出 `formatNumber(n)`、`formatDateTime(ts)`，内部用随
  `resolvedLocale` 重建的 `Intl.NumberFormat` / `toLocaleString(locale)`。
- 替换所有 `Intl.NumberFormat('en-US')` 与裸 `toLocaleString()` 调用点。

### Settings 语言项

- Appearance 区新增 Language 行（SettingRow），三段切换：Auto / English / 简体中文。
- 选项标签"English/简体中文"各自用目标语言显示（endonym），不随 UI 语言翻译。

### 后端 tag 翻译贯通

- 返回 `translated_name` 的端点新增 `lang` query param（默认 `zh-Hans` 保持兼容）：
  tags 列表、tag 更新、post tag-count facet、post 详情 tags（经 TagRepo 透传）。
- `translate_tag('…', 'en')` 直接返回 `None`（英文显示名即原始 tag 名，
  前端回退 `naturalizeTagName`）。
- 前端 API 调用统一带 `lang=resolvedLocale`；TanStack Query 的 queryKey 纳入 locale，
  切语言自动重取。
- 重新生成 API client（`pnpm genapi`，可用离线 schema dump）。

### 测试

- vitest：en/zh-Hans key 树深度一致性测试（运行时兜底，防 `as any` 绕过类型）；
  `pickLocale()` 纯函数单测（zh/en/其他/空列表）。
- 既有验证：`pnpm build`（vue-tsc）、`pnpm lint`、`pnpm test`；
  后端 `ruff check`、`pytest`（golden master 覆盖 repo 层改动）。

## 不做的事（YAGNI）

- 不引入 ja UI 目录（结构上加一个文件即可扩展；tag 层 `tag.ja.json` 已可用）。
- 不做 URL 中的 locale（单用户本地应用，无 SEO 需求）。
- 不做懒加载 locale chunk（两份目录体积小，不值得异步复杂度）。
